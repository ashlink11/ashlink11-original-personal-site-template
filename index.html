ABOUT THIS SITE

Accessibility for site visitors

Welcome to my portfolio. I am proud of the accessibility of this site, guided by the Mozilla Developer Accessibility docs as well as W3C's Web Content Accessibility Guidelines (WCAG).

I have written this page with screen-readers in mind. Screen readers provide accessibility to those with visual impairment. Scroll over some of the images to view the "alt text" that's behind the scenes.

Websites like this, designed in alignment with principles of "dark mode" and "flat UI" design are accessible because proper contrast levels reduce eye-strain. My color palette features low-saturation and harmonious colors of many variants and opacities. The contrast of text and background is never below 4.5:1, per WCAG. For more info, check out my color palette.

I carefully considered the accessibility of the fonts, type of styling, and number of elements per section of the page. Minimalism provides accessibility to those with cognitive differences.

For developers

This site is accessible, mobile-first and responsive, built from scratch. It's the 5th website I've built so far in 2020 and represents the synthesis of my front-end skills from the General Assembly Software Engineering Immersive course and self-study.

I chose the MIT License for this site so others can use this open-source code without any licensing issues. It is free (in the sense of 'freedom'): you are free to use this code for commercial use. More information on my tech and licensing is found in my portfolio's code repository.

I wrote semantic HTML 5 from scratch keeping in mind best-practices, clean-code and accessibility at every line.

My CSS is written in Sass, which is highly readable and helps with keeping things DRY. I tried to use as few CSS styles as possible and used descriptive class names.

I used as few Bootstrap classes, containers, and rows as possible to simplify my site's grid and styling system.
